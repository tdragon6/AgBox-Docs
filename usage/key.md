# 字段说明
此文档介绍了各个模块的字段含义和说明

## 收件箱
- `员工名称`：通知消息的来源，即数字员工/协调器名称
- `通知类型`：
    - `员工升级`：数字员工等级或品质升级时的通知消息
    - `任务完成`：任务完成时的通知消息，`finished`、`interrupt` 和 `failed` 等状态任务结束时均会通知
- `任务消息`：触发通知的任务输入消息
- `任务/升级状态`：`员工升级` 类消息分为 `员工等级` 和 `员工品质`，`任务完成` 类消息显示为对应 `任务状态`
- `是否已读`：初始未读，查看后更新为已读
- `创建时间`：通知消息创建时间
- `更新时间`：初始为创建时间，已读后更新时间

## 数字员工
- `员工名称`：数字员工的名称及其头像
- `员工描述`：数字员工描述，协调器会根据此描述判断是否委派任务给该数字员工
- `员工作者`：数字员工作者，读取时若不存在此字段，显示为 `Anonymous`，创建数字员工时自动设置为当前配置的用户名
- `员工部门`：数字员工所属部门
- `员工等级`：`L1` - `L9`，等级从左往右逐渐增加
- `员工品质`：`普通`、`高级`、`稀有`、`史诗` 和 `传说`，品质从左往右逐渐增加
- `创建时间`：数字员工创建时间
- `更新时间`：数字员工更新时间

## 任务管理
### 任务列表
- `任务消息`：任务的输入提示词消息
- `优先级`：`P1`、`P2`、`P3`、`P4`，优先级从左往右逐渐降低
- `触发来源`：
    - `user`：用户下发的任务
    - `scheduler`：定时器下发的任务
- `任务状态`：
    - `pending`：等待中（排队中的任务）
    - `running`：运行中
    - `finished`：已完成
    - `interrupt`：任务中断（一般为手动取消或关闭服务时清理流程取消的任务）
    - `failed`：任务失败
- `创建时间`：任务创建时为空，任务开始执行时更新为当下时间
- `更新时间`：任务更新时间
- `任务类型`：此字段在 `会话任务` 页面不显示，在 `会话任务` 页面也不支持搜索和排序
    - `会话任务`：会话类任务
    - `同步协调器`：同步协调器规划的项目类任务
    - `异步协调器`：异步协调器规划的项目类任务
- `员工名称`：执行任务时涉及的全部数字员工，此字段在 `会话任务` 页面不显示，在 `会话任务` 页面也不支持搜索和排序

### 任务消息
- `任务消息`：任务执行中的每一步消息
- `消息角色`：
    - `user`：用户/外部输入的消息
    - `assistant`：模型返回的结果类消息
    - `tool`：调用工具类消息
- `创建时间`：消息创建时间
- `推理内容`：模型推理内容

## 项目列表
- `项目名称`：项目名称
- `项目描述`：项目描述
- `项目涉及数字员工`：当前项目涉及的数字员工
- `项目历史涉及数字员工`：项目历史涉及的数字员工，包括已移除项目的数字员工
- `创建时间`：项目创建时间
- `更新时间`：项目更新时间

## 定时任务
- `任务名称`：任务名称
- `任务描述`：任务描述
- `员工名称`：参与定时任务的数字员工/协调器
- `挂载对象`：定时任务挂载的对象，项目或会话
- `任务消息`：定时任务执行任务时的输入提示词消息
- `优先级`：定时任务执行的优先级
- `项目任务`：该定时任务是否为项目任务
- `暂停`：定时任务是否处于暂停状态
- `定时时间`：定时任务的执行时间周期表达式，兼容 linux crontab 表达式，格式为 `* * * * *`，从左至右依次为秒、分、时、日、月、周
- `创建时间`：定时任务创建时间
- `更新时间`：定时任务更新时间
- `下次时间`：定时任务下一次执行的时间

## 技能列表
- `技能名称`：技能名称
- `技能描述`：技能描述，模型会根据此描述判断是否需要使用该技能
- `技能分类`：技能所在分类
- `是否存在脚本`：技能是否存在脚本

## 技能市场
- 表格字段：
    - `技能名称`：技能名称
    - `作者`：技能作者
    - `技能描述`：技能描述，模型会根据此描述判断是否需要使用该技能
    - `Github地址`：技能源Github仓库地址
    - `来源地址`：技能源地址
    - `点赞数`：技能点赞数
    - `更新时间`：技能更新时间
- 搜索字段：
    - `关键词`：技能搜索关键词
    - `技能分类`：技能所在分类，对应 `skillsmp` 的 `category` 字段
    - `职业分类`：技能所属职业分类，对应 `skillsmp` 的 `occupation` 字段
- 各字段参考：[skillsmp api 文档](https://skillsmp.com/docs/api){target="_self"}
